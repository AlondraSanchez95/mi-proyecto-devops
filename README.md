Descripcion del proyecto:
En este pequeno proyecto se esta simulando un flujo DevOps con GitHub.

Objetivo:
Saber subir cambios y mejoras de codigo a GitHub mediante commits y pull request. Ademas del uso de repositorios publicos de GitHub y uso de ramas para la creacion de codigo en entornos aislados.

Estructura:
mi-proyecto-devops es la carpeta principal
src es la carpeta donde se encuentra el codigo del sitio web
scripts es la carpeta en donde se encuentra el simulador de despliegue
tambien se tiene un README (donde estoy escribiendo esto) y un git ignore.

Flujo de trabajo:
Primero se creo el repositorio con la linea principal, despues se creo la estructura basica tales como las carpetas separadas, el git ignore y el archivo README.
Despues se creo el HTML y el simulador de despliegue sh.
Ya para finalizar inclui mejoras en el estilo del HTML con el CSS y una que otra cosa mas en el HTML.

Comandos Git utilizados:
Clone: para clonar el repositorio en mis archivos locales
Add: para seleccionar que subir a la nube
Commit: para guardar y crear un registro de lo subi localmente
Push: para ahora si subirlo a la nube especificando de donde viene
Checkout: para moverse entre las ramas existentes o crearlas si no existen

Simulador de despliegue:
Para el simulador solo use el Git Bash de Visual Studio Code
