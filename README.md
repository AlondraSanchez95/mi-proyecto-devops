# Mi Proyecto DevOps

## Descripción
Proyecto web estático para practicar control de versiones y flujo DevOps utilizando Git y GitHub.

## Objetivo
Aplicar conceptos básicos de DevOps como:
- Control de versiones
- Uso de ramas
- Simulación de despliegue

## Estructura del proyecto
mi-proyecto-devops/
│── src/
│   ├── index.html
│   ├── styles.css
│── scripts/
│   └── deploy.sh
│── README.md
│── .gitignore

## Flujo de trabajo
1. Se creó el repositorio en GitHub
2. Se clonó en local
3. Se realizó un primer commit con la estructura
4. Se agregó un script de despliegue
5. Se trabajó con una rama (feature-update)
6. Se subieron los cambios al repositorio

## Comandos Git principales
- git init
- git clone
- git add
- git commit
- git push
- git checkout -b
